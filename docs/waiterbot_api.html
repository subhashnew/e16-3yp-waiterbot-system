Middlewares

Middlewares permits communication and management for the backend by operating as a hidden translation layer. By using express.js framework we can easily manage middlewares as we need. Types of middlewares we use in our backend design are listed below.

  1. Application layer middlewares

    • Logging middleware

  2. Application layer middlewares

Design Decisions

Following aspects where considered when designing the API to increase the quality of the API.

  1. Version controlling

    Version control comes in handy when we need to have some changes in our API. Existing clients may not like to update their applications when we change the API. A versioning strategy allows clients to continue using the existing API and migrate their applications to the newer API when they are ready.

  2. Input validation

    Using the joi package validates the input first.

  3. Server jurnal

    Winston / log4js

  4. Security

NodeJS

Node js is an event-driven environment which has a single threaded event loop as well as async handlers. Because of it’s event loop and async handlers, it has less context switching compared to multi threaded servers.Since our backend has no heavy computations, we have selected Node.js as the WaiterBot system's backend technology.

MongoDB

MongoDB is a flexible document based database which has great query performance. It has highly availability and scalability. By using MongoDB we can develop fast and can move fast.

AWS

To have better scalability and reliability we use AWS S3 storage as our main storage. All the static content will be stored in the S3 bucket and CDN will access the S3 bucket to deliver the content fast to the users.
