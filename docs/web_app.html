Design and Progress

Web Application

WaiterBot web applications is designed for the owner of the restaurant. This gives the entire details of the menu, table, robots, orders and store. Owner of the restaurant can easily update the menu with steps. When compared to the traditional menu card ordering system, the updating process is quite easier. This helps the customers have a good experience at the restaurant by getting more updated information about the food items thereby increasing the reputation of the restaurant.

If the table layout of the restaurant needs to be changed, it can be changed according to the requirement of the restaurant staff and the table details have to updated accordingly.

UI/UX and Other Features

  • Since the web app is for monitoring the restaurant and updating menu items. The web application is designed prioritizing the restaurant management.
  • Web application has a responsive design so that it can be views from any device.
  • Real time notifications

ReactJS

ReactJS was used as the frontend development framework. We chose ReactJS because, by using ReactJS we can web applications with dynamic content, development is fast, can build scalable web applications due to its DOM rendering architecture and also ReactJS has large community support with vast number of libraries.

Redux

Redux was used as the state manager. With Redux we can store all the state information in a single store thereby providing a single source of truth. This helps in rapid development and also when debugging.

Axios

Axios library was used for making HTTP requests from frontend web application to the backend server.

Socket.io

Socket.io was used for realtime push notifications.
