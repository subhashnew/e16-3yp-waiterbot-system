Design and Progress

Desktop Application

WaiterBot desktop application is designed for the operator of the robots. Accepting orders, deploying the robots after orders are ready, changing the availabilty of the food items can be done using this application.

When the desktop appllication is launched, it will start a local MQTT broker. Robots and the desktop application will be communicating via this MQTT broker. Robots will be connected to the broker using MQTT over TCP and desktop application will be connected to the broker by using MQTT over Web Scockets.

UI/UX and Other Features

  • Since the main purpose of the desktop app is the order management, the desktop app is designed prioritizing the order management.
  • Real-time notifications when new orders are placed and robot sends alerts
  • Support cross platforms

ReactJS

ReactJS was used as the frontend development framework. We chose ReactJS because, by using ReactJS we can web applications with dynamic content, development is fast, can build scalable web applications due to its DOM rendering architecture and also ReactJS has large community support with vast number of libraries.

Electron

Electron was used for building the desktop application. Main reason for chosing Electron was because we can build cross platform desktop applications and the development can be done using web technologies HTML, CSS and JavaScript.

Redux

Redux was used as the state manager. With Redux we can store all the state information in a single store thereby providing a single source of truth. This helps in rapid development and also when debugging.

Axios

Axios library was used for making HTTP requests from frontend web application to the backend server.

Socket.io

Socket.io was used for realtime push notifications.
